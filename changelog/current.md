# Changelog (Unreleased)

Record image-affecting changes to `manager/`, `worker/`, `copaw/`, `openclaw-base/`, `hiclaw-controller/`, and release-facing `install/` / Helm chart changes here before the next release.

---

- feat(qwenpaw): add the QwenPaw worker runtime Python package baseline with runtime config sync, storage sync, heartbeat reporting, Matrix channel overlay, and focused unit tests.
- fix(controller): surface Kubernetes Pod container failures in Worker backend status and status API responses.
- feat(controller): expose low-cardinality AgentTeams controller metrics and optional Helm ServiceMonitor.
- feat(controller): add Matrix AppService Human SSO identity provisioning with hash-derived Matrix IDs, AppService login, deletion deactivation, and Team admin/member identity resolution from Human status.
- fix(agent): update file-sharing path guidance for CoPaw and Team Leader agents to use `/root/hiclaw-fs/agents/...` instead of the retired `/root/.hiclaw-worker/...` path.
- feat(helm): add a Helm LLM preflight hook and reusable `hiclaw llm-preflight` command to validate API key/base URL/model before controller startup.

- fix(copaw): harden Matrix channel control-command handling, task-thread routing, NO_REPLY suppression, and cancellation noise handling.
- feat(controller): add OpenKruise Sandbox backend support for Workers via `spec.backendRuntime=sandbox`, including SandboxClaim lifecycle, status watches, CRD schema, and Helm RBAC/env wiring.
- fix(controller): materialize sandbox Worker runtime env/auth material into worker-deps storage before creating SandboxClaim deps and block legacy pod-to-sandbox runtime switches until the Worker is stopped.
- feat(controller): add per-agent `spec.resources` support for Manager, Worker, Team Leader, and Team Worker CRDs.
- fix(worker): pass `X-HiClaw-Cluster-ID` when remote Workers refresh controller-issued STS credentials for OSS and Nacos AI registry access.
- feat(hiclaw-controller): support a separate agent-pod-template for `deployMode: Remote` Workers via an optional `pod-template-remote.yaml` key on the controller-scoped pod-template ConfigMap; remote-mode Pod creation prefers this key and transparently falls back to `pod-template.yaml` when it is absent or empty, while non-remote/Sandbox paths keep ignoring it.
- feat(controller): move Kubernetes resources to the `agentteams.io/v1beta1` API group and decouple Team membership through `spec.workerMembers` references to standalone Worker CRs.
- feat(runtime): let Worker runtimes consume terminal `AGENTTEAMS_*` storage, controller, and auth environment variables while preserving existing `HICLAW_*` deployment compatibility.

- **OpenHuman runtime**: OpenHuman added as the fourth Worker runtime with native Matrix support via `channel-matrix` feature flag; includes controller routing (K8s + Docker backends), Dockerfile, entrypoint script, agent template, Helm chart integration, and Makefile build targets.
- **Multi model providers**: Worker, Team, and Manager specs can now select a Higress model provider via `spec.modelProvider`; the controller resolves the provider, injects the matching gateway URL into runtime config, and authorizes consumers only on the selected AI route.
- **modelProvider authorization boundary**: Controller reconcilers now own provider-specific AI route authorization, while provisioning keeps using the default gateway authorization path to avoid duplicate provider coupling.
- **Matrix AppService mode**: The controller can register as a Matrix Application Service and provision/log in users with the `as_token` instead of per-user passwords (legacy password auth is preserved when disabled). Enabled by default via `AGENTTEAMS_MATRIX_APPSERVICE_ENABLED`; the install script and the Helm `runtime-env` Secret generate and persist `AGENTTEAMS_MATRIX_APPSERVICE_AS_TOKEN` / `AGENTTEAMS_MATRIX_APPSERVICE_HS_TOKEN`. Set `AGENTTEAMS_MATRIX_APPSERVICE_USER_NAMESPACE_REGEX` to narrow the exclusive user namespace when running against a shared / pre-existing homeserver.
- **SandboxSet prewarm sizing**: Sandbox-backed deployments can configure the unified SandboxSet warm pool with `AGENTTEAMS_SANDBOX_PREWARM_SIZE` / `controller.sandbox.prewarmSize` (default `1`, `0` disables warm replicas), and SandboxSet container requests now match limits from `AGENTTEAMS_K8S_WORKER_CPU` / `AGENTTEAMS_K8S_WORKER_MEMORY`.
- **Team CRD break-change cleanup**: Team CRDs now accept `spec.workerMembers` references only; legacy inline `spec.leader` / `spec.workers` schema, API compatibility paths, auto-migration wiring, migration annotations, and migration controller tests were removed from the controller and Helm chart.
- **Team inline compatibility cleanup**: Registry migration now creates independent Worker CRs and Team `spec.workerMembers` references, the obsolete `hiclaw migrate` CLI is removed, decoupled Team phase reflects Team convergence separately from Worker runtime readiness, and historical migration finalizers are cleared during Team deletion.

**Bug Fixes**

- **CI image aliases and Worker CRD schema**: Local image build targets now keep legacy `hiclaw/*` aliases alongside AgentTeams image names so base-branch PR workflows can save/load artifacts, and Worker CRDs again expose existing lifecycle, runtime, credential, and sandbox volume spec fields including `spec.state`.
- **Manager-disabled controller startup**: Setting `AGENTTEAMS_MANAGER_ENABLED=false` now skips Manager CR informers, reconciliation, and CR-count listing so controller deployments can run in clusters where `managers.agentteams.io` is not installed.
- **Remote runtime auth routing**: Remote pod Workers keep `AGENTTEAMS_CLUSTER_ID` as the only remote TokenReview routing signal, while Sandbox worker dependency env no longer injects `AGENTTEAMS_CLUSTER_ID` or `AGENTTEAMS_AUTH_MODE`; QwenPaw heartbeat, STS refresh paths, OSS credential helpers, and find-skills scripts now send `X-AgentTeams-Cluster-ID` only when `AGENTTEAMS_CLUSTER_ID` is present.
- **SandboxClaim completed status**: Sandbox-backed Workers now treat a completed `SandboxClaim` as claim progress only and resolve the actual Sandbox by Worker labels before reporting runtime state, so Remote sandbox Workers no longer remain `containerState=unknown` after the target Sandbox is Running.
- **SandboxClaim template pruning**: Sandbox-backed Workers no longer send unsupported `SandboxClaim.spec.template`; pod-template metadata labels and annotations are mapped into SandboxClaim fields while PodSpec settings remain pod-backend-only, eliminating API-server `unknown field "spec.template"` warnings on ACS.
- **Matrix room diagnostics**: When Matrix `createRoom` is rejected, the controller now logs the token source, `whoami` sender, expected admin Matrix ID, room alias, invite list, and configured power levels without exposing access tokens, so room-creation permission failures can be diagnosed from controller logs.
- **Sandbox multi-instance status**: Sandbox-backed Workers and Managers now treat `SandboxClaim` as the authoritative binding before falling back to actual Sandbox label scans, so provider pools that temporarily expose multiple matching Sandboxes no longer produce `multiple sandboxes match` reconcile errors or unsafe Manager recreates.
- **Worker spec-change cleanup**: Worker and Team member container reconciles now delete any observable stale backend resource before recreating after a pod/runtime-affecting spec change, so sandbox `Claiming`, `Completed`, `Starting`, and multi-Sandbox states no longer block image updates from clearing the old runtime.
- **Team spec validation**: Teams without `workerMembers` now fail fast with a clear status message instead of entering provisioning with an empty member list.
- **Member runtime config isolation**: QwenPaw runtime config now lives only under `agents/<worker>/runtime/runtime.yaml` instead of the shared worker directory, and worker background sync skips that controller-owned file.
- **Decoupled Team lifecycle**: Decoupled Team members now inherit `leader.workerIdleTimeout` into their standalone Worker CRs unless an explicit member `idleTimeout` is set, and Team Room @mentions can wake sleeping decoupled Workers by patching the referenced Worker CR back to `Running`.
- **Legacy Team channel policy**: Legacy Team reconciliation now writes final Matrix channel allow-lists to member runtime config and re-adds the Team Leader to the Manager allow-list so controller integration tests observe durable policy state.
- **Sandbox worker-deps hardening**: Sandbox-backed Workers now prepare controller-owned worker-deps env/token/data material before claim creation, recycle stale SandboxClaims and bound Sandboxes on runtime-affecting changes, and use bounded ServiceAccount token projection for built-in SandboxClaim mounts.
- **CLI AgentTeams auth env**: The `hiclaw` CLI now discovers `AGENTTEAMS_CONTROLLER_URL`, `AGENTTEAMS_AUTH_TOKEN`, `AGENTTEAMS_AUTH_TOKEN_FILE`, and `AGENTTEAMS_CLUSTER_ID` while preserving legacy `HICLAW_*` fallbacks, so Manager and Worker containers can use the terminal env names for controller calls.
- **CoPaw worker runtime environment**: CoPaw workers now prefer AgentTeams storage/runtime environment variables while preserving legacy HiClaw fallbacks, and Qwen-style model health preflights disable thinking for lightweight readiness checks.
- **CoPaw Worker heartbeat**: CoPaw worker templates now seed heartbeat at a 10-minute interval so Team Leader agents created from the worker template can run heartbeat turns without requiring an explicit Team CR heartbeat spec.
- **Helm CRDs**: Removed unsupported `propertyNames` schema fields from Worker and Team CRDs so Kubernetes API servers accept the chart CRDs.
- **CoPaw local runtime paths**: CoPaw direct-run defaults now honor `COPAW_INSTALL_DIR` and `COPAW_WORKING_DIR` before falling back to local home-directory paths, while container entrypoints can continue to pass explicit directories.
- **CoPaw worker replies**: CoPaw's no-text debounce only recognized copaw `Content` objects, but the custom Matrix channel builds plain-dict content parts, so every message was buffered and the agent never replied. The channel now also recognizes dict-based text/audio parts; workers and team leaders reply as expected.
- **Worker Matrix token refresh**: `POST /api/v1/credentials/matrix-token` was registered against the `credentials` resource kind, but the worker credentials branch only permitted `ActionSTS`, so the route was dead and workers could not self-refresh on a homeserver 401. The self-scoped credentials branch now also permits `ActionRefreshMatrixToken` (and the misplaced dead entry was removed).
- **Helm AppService tokens**: The Helm `runtime-env` Secret now generates and preserves the AppService `as_token` / `hs_token` (values override -> existing Secret -> generated) so a default Helm/in-cluster install no longer crash-loops the controller, and a template comment trim-marker that broke `helm lint` YAML parsing was fixed.
- **Worker room reuse**: Freshly-created Workers now release a stale Matrix worker-room alias before joining, preventing a new same-name Worker from reusing the previous Worker's DM room.
- **Worker runtime cleanup**: Deleting a Worker now removes its agent-scoped runtime config, and deleting a decoupled Team rewrites retained member Workers back to standalone runtime context so same-name Workers do not inherit stale Team facts.
- **Team room archival**: Deleting a Team now renames preserved Matrix Team and Leader DM rooms with a `[deleted]` suffix before releasing their aliases, making archived rooms distinguishable from active rooms.
- **Decoupled Team deletion**: Deleting a decoupled Team now uses the recorded leader runtime name when the leader Worker CR is already gone, ensuring the archived Leader DM name and released room alias still target the correct Matrix room.
- **Worker config updates**: Updating a Worker's `model` or `mcpServers` now rewrites the corresponding OSS configuration files without recreating the Worker container; image/runtime/env-style container changes still trigger recreation.
- **Worker MCP config merge**: Worker `mcpServers` updates now replace only the current gateway-managed MCP entries in `config/mcporter.json` while preserving existing external MCP entries in the Worker's storage directory.
- **Remote-managed local workers**: Edge Worker CRDs and Team runtime config paths now support LoongSuite-managed local Claude Code workers, including runtime-neutral `MemberRuntimeConfig` delivery, remote heartbeat status, and AgentSpec package overlay for Claude Code skills/context.
- **Remote-managed OpenClaw local workers**: Added the LoongSuite-managed OpenClaw local runtime bundle, Node worker, MCP bridge, AgentSpec prompt/skill packaging, and remote integration tests aligned with the Claude Code Node runtime path.
- **Remote-managed OpenClaw model proxy**: OpenClaw local workers now pass through streaming OpenAI chat-completion responses from the model gateway so OpenClaw no longer turns SSE chunks into `LLM request failed`.
- **Remote-managed OpenClaw trace**: OpenClaw local bundles now ship the loongsuite-js `opentelemetry-instrumentation-openclaw` plugin and bridge local loongsuite-pilot CMS/OTLP configuration into the OpenClaw child process without persisting trace credentials in `openclaw.json`.
- **Remote-managed local model mode**: Claude Code and OpenClaw local workers now support `modelConfigMode=native-config`, keeping TeamHarness prompt/skill/MCP management while leaving model selection and credentials to the runtime's native local configuration.
- **Edge Worker native model config**: Edge Worker CRs may set `spec.model` to `native-config`; the controller keeps that value on the CR for cloud-side visibility but omits `desired.model` from runtime.yaml so local workers can rely on their native model configuration.
- **Remote-managed local model defaults**: Claude Code and OpenClaw local workers now default to `modelConfigMode=native-config`; managed model injection remains available when explicitly configured with `managed-runtime` or `managed-global`.
- **Remote-managed local model credentials**: Claude Code and OpenClaw local workers now disable the local broker `/v1/credentials/model` endpoint in `modelConfigMode=native-config` while keeping runtime context, Matrix, storage, and skill-registry broker endpoints available for TeamHarness MCP tools.
- **Remote-managed Claude Code parity**: Claude Code Node local workers now inject the shared team prompt, filter role-scoped TeamHarness skills in runtime overlays, and always pass the default `bypassPermissions` permission mode unless explicitly overridden.
- **Remote-managed local runtime bundles**: Claude Code and OpenClaw local runtime bundle tarballs now use versioned AgentTeams package names, starting with `agentteams-*-local-runtime-0.0.1.tar.gz`, while keeping the LoongSuite runtime template IDs stable.
- **Remote-managed Claude Code worker cleanup**: The retired Python Claude Code local worker package and its pytest suite have been removed; Claude Code local runtime delivery now uses the Node worker path only.
- **Edge Worker bootstrap JWT**: Edge Worker bootstrap tokens now carry a controller-signed JWT instead of the raw worker UUID; the controller verifies the JWT against a controller-owned signing Secret and issues short-lived ServiceAccount tokens from `/api/v1/edge/token`.
- **Remote-managed Edge token refresh**: Claude Code and OpenClaw local Node workers now refresh the Edge ServiceAccount token before it expires by re-exchanging the persisted bootstrap JWT, and persist the refreshed JWT returned by the controller for restart continuity.
- **Remote-managed Matrix session commands**: Claude Code and OpenClaw local workers now handle exact `/stop` and `/resume` Matrix commands; `/stop` aborts the active room task while preserving the current session, while `/resume` aborts the active task and clears/rotates stored room context without delivering the command to the agent runtime.
- **Remote-managed Node Matrix scheduler**: Claude Code and OpenClaw local workers now durably stage Matrix events before advancing the sync token, merge same-room pending messages into a single turn, run different rooms concurrently up to a bounded limit, and bootstrap first-run history without replaying historical tasks or control commands.
- **Remote-managed Node Matrix rendering**: Claude Code and OpenClaw local bundles now ship `markdown-it` with the shared Node runtime core so Matrix replies render Markdown headings, tables, lists, inline code, and emphasis through `formatted_body`.
- **Remote-managed Edge bootstrap JWT**: Claude Code and OpenClaw local Node workers now parse `jwtToken` bootstrap payloads, exchange them with `/api/v1/edge/token`, and persist refreshed bootstrap JWTs back to the original private token file for restart continuity.
- **Remote-managed local package view**: LoongSuite-managed local Claude Code workers now mirror AgentSpec package Soul, AGENTS, and skills back to the OSS worker workspace view so the console can display and edit custom-worker package content.
- **Edge Worker heartbeat status**: Edge Workers now reconcile every minute and fall back from `Running` to `Pending` when heartbeats stop for more than two minutes, without touching container state or ordinary Worker reconcile cadence.
- **Worker access entries updates**: Updating Worker or Team member `accessEntries` now only changes credential scope resolution and no longer recreates the underlying Worker Pod.
- **Sandbox Worker image updates**: Updating a sandbox-backed Worker image now deletes the old `SandboxClaim` and bound `Sandbox`, waits for both to disappear, and only then creates the replacement claim; SandboxSet workers now use a zero-second termination grace period to speed recycle.
- **Sandbox RRSA CredentialProviders**: Sandbox worker-deps RRSA now ensures the remote target namespace before creating `agentteams-env`, `agentteams-token`, and `agentteams-data` CredentialProviders in that namespace, matching the AgentIdentity CRD scope and avoiding missing-resource errors during reconcile.
- **Sandbox runtime image and claim shape**: SandboxSet now uses the regional ACS `agent-runtime:v0.0.9` image while SandboxClaim carries the business worker image, keeps Pod-like claim names, and emits worker/manager identity plus `security.agents.kruise.io/agent-name=agentteams` through `spec.labels`.
- **SandboxClaim ACS spec fields**: Built-in worker-deps dynamic mounts now use `attributes.credentialProviderName`, SandboxClaim sets `waitReadyTimeout: 2m`, and the controller continues omitting `skipInitRuntime`.
- **CoPaw worker replies**: CoPaw's no-text debounce now recognizes dict-based text/audio parts emitted by the custom Matrix channel, so messages are not incorrectly buffered forever and Workers / Team Leaders reply as expected.
- **Worker Matrix token refresh**: Worker self-scoped credentials now allow Matrix token refresh, making `POST /api/v1/credentials/matrix-token` usable when Workers recover from homeserver 401 responses.
- **Helm AppService tokens**: The Helm `runtime-env` Secret now generates and preserves AppService `as_token` / `hs_token` values so default Helm installs can start the controller in AppService mode without crash loops.
- **Human SSO spec preservation**: Human finalizer add/remove now uses merge-patch instead of full Update, so `spec.identitySource` is not wiped when the controller first reconciles an external-SSO Human CR.
- **Human identity-switch guard**: Removing `spec.identitySource` from an already-provisioned SSO Human (or otherwise changing the derived Matrix user ID) no longer silently re-provisions a new legacy account while leaving `status.rooms` pointing at the old user's memberships — which left the Human reporting Active with rooms while the new identity was in none of them. The reconciler now degrades the Human and asks the operator to recreate the CR for any post-provisioning MXID change, in both the SSO→legacy and legacy→SSO directions.
- **Human status.identitySourceFingerprint removed**: The write-only `status.identitySourceFingerprint` field (set for SSO Humans but never read) has been dropped from the Human API, controller logic, and `config/crd` schema. It was missing from the Helm CRD, so every SSO Human reconcile logged a harmless `unknown field "status.identitySourceFingerprint"` API-server pruning warning; removing the field eliminates the warning. `status.displayNameSyncedGeneration` is kept.
- **AppService startup race logs**: During controller startup, Worker/Human reconciles could call the Matrix AppService register/login API before cluster init finished registering and verifying the controller's `as_token`, producing a burst of `error`-level "AppService register rejected by homeserver" / "Reconciler error" logs (HTTP 401 `M_UNKNOWN_TOKEN`) with stacktraces until the token became active. The Matrix client now recognizes this transient `M_UNKNOWN_TOKEN` case as a retryable `ErrAppServiceNotReady`, logs it at `info`, and the Worker and Human reconcilers requeue quietly with a short backoff instead of surfacing a hard error.
- **Team AppService not-ready handling**: The Team member reconcile path only inspected the error from `ReconcileMemberInfra` and ignored its `reconcile.Result`, so when Matrix infra signaled the transient `ErrAppServiceNotReady` startup race (short requeue, `error=nil`) a Team member would treat infra as successful and continue running later phases (ServiceAccount/config/container), even marking the member observed before Matrix/Gateway/Room were provisioned. The Team path now detects the short requeue, stops before later phases, keeps the member `Pending`, and requeues quickly with the same quiet backoff as the Worker path.
- **Team identity for SSO Humans**: Team admin login, room invites, coordinator power levels, channel policies, and the runtime roster now resolve every referenced Human's Matrix ID from the Human CR's provisioned `status.matrixUserID` instead of assuming `localpart == name`. Previously an SSO team admin (whose Matrix localpart is a hash) was authenticated as a phantom `@username` AppService user and SSO human members were invited under the wrong `@name` ID, so SSO and password-authenticated users behaved differently. Both now resolve to the same authoritative identity; the spec `matrixUserId` is validated against it, and an unprovisioned SSO admin is rejected rather than resolved to the wrong account.
- **AgentTeams storage alias**: Controller, Manager, Worker, CoPaw Worker, and local MinIO mirror scripts now derive the mc alias from the AgentTeams storage prefix, so `agentteams/agentteams-storage` is treated as object storage instead of a local path.
- **Worker AgentTeams envs**: Controller-created Workers now receive and consume `AGENTTEAMS_*` runtime, credential, storage, Matrix, and gateway environment variables.
- **AgentTeams CLI envs**: The `hiclaw` CLI now reads `AGENTTEAMS_DEFAULT_MODEL` and `AGENTTEAMS_NACOS_REGISTRY_URI` for worker defaults and package shorthand expansion.
- **AgentTeams runtime contracts**: Controller config, Manager/Worker runtimes, local install defaults, agent skills, test fixtures, and default image names now use `AGENTTEAMS_*`, `agentteams.io`, and `agentteams/*` contracts without `HICLAW_*` compatibility fallbacks.
- **QwenPaw Nacos CLI**: QwenPaw Worker images now pin `@nacos-group/cli` to `1.1.2`.
- **QwenPaw Worker startup**: QwenPaw Worker images now install the built-in TeamHarness and WorkerFlow plugins during Docker build, so normal startup and `runtime.yaml` reapply prepare image-bundled plugins without rerunning `qwenpaw plugin install`.
- **QwenPaw Worker startup**: QwenPaw Worker now keeps QwenPaw's built-in QA agent profile disabled so the worker starts only the managed default workspace instead of an extra QA workspace.
