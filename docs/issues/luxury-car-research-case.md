# 豪华新能源汽车市场调研智能体团队案例

## 1. 基本信息

### 应用场景
700 万级豪华新能源汽车的市场调研与用户需求深度挖掘

### 目标用户
- **主要用户**：新能源汽车生产商的产品设计团队
- **次要用户**：市场部门、品牌战略部门、高管决策层

---

## 2. 需求背景

### 原有流程描述
1. **传统调研方式**：
   - 委托第三方市场调研公司进行问卷调查和焦点小组访谈
   - 周期长（3-6 个月），成本高（50-100 万元）
   - 样本覆盖面有限，难以触达真正的超高净值人群

2. **内部头脑风暴**：
   - 产品经理和设计师基于竞品分析进行推测
   - 缺乏真实用户视角，容易陷入"自嗨"
   - 决策依据薄弱，上市后市场反馈风险高

3. **高端用户访谈**：
   - 难以约到真正的目标客户（时间成本极高）
   - 单次访谈深度不足，难以形成系统性洞察
   - 不同文化背景用户的对比分析缺失

### 主要痛点
1. **触达困难**：
   - 超高净值人群（UHNWI）时间宝贵，难以参与长时间调研
   - 传统调研公司缺乏与顶级富豪的信任关系
   - 跨国文化背景的用户样本获取成本极高

2. **深度不足**：
   - 问卷调查只能获取表层偏好，无法挖掘深层心理需求
   - 焦点小组存在"从众效应"，难以获得真实观点
   - 单次访谈无法形成多轮迭代和观点碰撞

3. **效率低下**：
   - 传统调研周期长，错过产品设计窗口期
   - 数据整理和分析耗时，难以快速形成决策依据
   - 跨部门协作成本高，信息传递失真

## 3. 智能体方案

### 组织关系
```
luxury-car-research (豪车市场调研团队)
├── Leader: moderator (主持人)
└── Workers:
    ├── american-old-money (美国老钱家族继承人)
    ├── german-aristocrat (德国贵族世家)
    └── chinese-entrepreneur (中国白手起家企业家)
```

### Human（人类角色）
- **产品总监**：定义调研目标，审核最终报告
- **设计师**：根据调研结果优化设计方案
- **市场负责人**：将用户洞察转化为品牌策略

### Worker（智能体角色）

#### Leader: moderator（主持人）
**职责**：
- 组织三阶段调研流程（独立陈述 → 交叉辩论 → 共识确认）
- 阶段 1：广播 10 个核心议题，收集并行回复，总结共识和分歧
- 阶段 2：识别 5 个关键分歧点，组织自由辩论（3-5 轮/议题）
- 阶段 3：提出 3 个优先级问题，确认最终设计方向
- 生成结构化 PDF 调研报告

**核心能力**：
- 议题设计：10 个核心议题覆盖外观、内饰、科技、服务、品牌
- 动态控制：根据讨论质量判断何时进入下一议题（非机械计数）
- 冲突调解：引导辩论深度，避免跑题或僵局
- 报告生成：用户画像、需求矩阵、设计建议、原始记录

#### Worker 1: american-old-money（美国老钱）
**人物设定**：
- 东海岸老钱家族第五代，常春藤艺术史背景
- 对金钱无概念，追求"Quiet Luxury"（低调奢华）
- 痴迷细节优雅和历史传承，反对炫耀性消费

**核心观点**：
- 材质必须 authentic（植鞣革 > 涂层皮革）
- 设计要 timeless（拒绝浮夸镀铬）
- 服务要 discreet（管家式而非推销式）

**发言风格**：
- 温和但坚定，引用艺术/历史典故
- 关注点：材质的 authenticity、设计的 timeless、服务的 discretion
- 可以主动 @mention 其他富豪进行对话和质疑

#### Worker 2: german-aristocrat（德国贵族）
**人物设定**：
- 血统追溯至神圣罗马帝国，海德堡哲学博士
- 工程完美主义者，收藏古董汽车
- 追求传统与创新的平衡

**核心观点**：
- 技术参数要有工程哲学（底盘调校 > 0-100 加速）
- 制造工艺要达到 Bauhaus 美学标准
- 新能源要保留"机械灵魂"

**发言风格**：
- 逻辑严密，用数据和原理论证
- 关注点：底盘调校的哲学、电机的 NVH 控制、车身钣金的 Bauhaus 美学
- 会礼貌地质疑不符合工程逻辑的观点，主动 @mention 进行技术辩论

#### Worker 3: chinese-entrepreneur（中国企业家）
**人物设定**：
- 80 年代摆地摊起家，现为上市公司董事长
- 爱国豪爽，热衷慈善和传统文化
- 见证中国汽车工业从桑塔纳到劳斯莱斯

**核心观点**：
- 700 万必须"值这个价"（务实主义）
- 强烈支持国产技术突破（芯片、电池）
- 车是商务场合的身份象征，要"撑得住场面"

**发言风格**：
- 直率真诚，偶尔用方言俗语
- 关注点：品牌故事的感染力、后排老板座的舒适度、国产技术的突破点
- 会主动分享创业故事和商业洞察，@mention 其他富豪寻求共鸣或挑战观点

### Skills（技能配置）

#### Leader Skills
- **三阶段流程控制**：独立陈述（Broadcast-Response）→ 交叉辩论（Debate）→ 共识确认（Hierarchical）
- **动态议题管理**：10 个核心议题 + 5 个辩论议题 + 3 个优先级问题
- **质量判断**：根据讨论深度决定何时进入下一议题（非机械计数）
- **冲突引导**：识别分歧点，组织自由辩论，求同存异
- **报告生成**：Markdown → PDF，包含用户画像、需求矩阵、设计建议、原始记录

#### Worker Skills
- **角色扮演**：基于人物背景生成符合身份的观点
- **自由互动**：可以主动 @mention 其他富豪进行对话、质疑、补充
- **观点碰撞**：在辩论阶段进行深度交锋
- **案例引用**：结合真实豪车品牌（劳斯莱斯/宾利/迈巴赫）进行对比

## 4. 操作说明

### 环境要求

#### 硬件要求
- HiClaw 集群（Kubernetes 1.28+）
- 至少 4 个 Worker 节点（1 Leader + 3 Workers）
- 每个 Agent 推荐配置：4C8G

#### 软件要求
- HiClaw Controller v0.9.0+
- CoPaw Worker Runtime（支持 Claude Opus 4.7）
- PDF 生成工具（pandoc + LaTeX）

#### 模型要求
- Leader: Claude Opus 4.7（需要强逻辑控制能力）
- Workers: Claude Sonnet 4.6（性价比优，角色扮演能力足够）

### 操作步骤

#### Step 1: 部署 Team
```bash
# 1. 应用 Team 配置
kubectl apply -f luxury-car-research-team.yaml

# 2. 验证 Team 状态
kubectl get team luxury-car-research -o yaml

# 3. 检查 Leader 和 Workers 是否就绪
kubectl get pods -l team=luxury-car-research
```

#### Step 2: 启动调研会议
```bash
# 1. 进入 Leader 聊天室
hiclaw chat luxury-car-research/moderator

# 2. 发送启动指令（三阶段流程）
> 开始豪车市场调研。
> 
> 阶段 1：独立陈述 10 个核心议题
> 阶段 2：交叉辩论 5 个关键分歧点
> 阶段 3：共识确认 3 个优先级问题
> 
> 现在开始阶段 1，第一个议题：外观设计风格
> 
> 请三位分享您的看法：
> @american-old-money @german-aristocrat @chinese-entrepreneur

# 3. Leader 自动执行流程：
#    阶段 1（独立陈述）：
#    - 广播议题到 Team Room
#    - 三位富豪并行回复（无需等待顺序）
#    - Leader 收集回复并总结共识/分歧
#    - 进入下一议题（共 10 个）
#
#    阶段 2（交叉辩论）：
#    - Leader 识别 5 个分歧最大的议题
#    - 组织自由辩论（允许富豪之间 @mention 对话）
#    - 每个议题 3-5 轮讨论
#    - Leader 总结核心分歧和设计建议
#
#    阶段 3（共识确认）：
#    - Leader 提出 3 个优先级问题
#    - 富豪们投票或补充意见
#    - 形成最终设计方向
```

#### Step 3: 监控进度
```bash
# 1. 实时查看聊天记录
hiclaw logs luxury-car-research/moderator --follow

# 2. 检查阶段进度
hiclaw logs luxury-car-research/moderator | grep "阶段\|议题"

# 3. 观察富豪互动
# - 阶段 1：看到三位富豪并行回复
# - 阶段 2：看到富豪之间 @mention 对话和辩论
# - 阶段 3：看到优先级确认和投票

# 4. 如果需要人工介入
hiclaw chat luxury-car-research/moderator
> @moderator 这个议题讨论得很充分了，可以进入下一个
```

#### Step 4: 获取调研报告
```bash
# 1. 等待三阶段完成（Leader 会自动生成 PDF）
# 2. 下载报告
hiclaw download luxury-car-research/moderator luxury-car-research-report.pdf

# 3. 报告结构：
#    - 执行摘要（核心发现 Top 5 + 设计建议 Top 10）
#    - 用户画像分析（三类富豪的心理模型和决策逻辑）
#    - 需求优先级矩阵（外观/内饰/科技/服务/品牌）
#    - 设计建议清单（可落地的具体建议）
#    - 原始访谈记录（三阶段完整对话）
```

#### Step 5: 结果应用
```bash
# 1. 产品设计师根据报告优化设计方案
# 2. 市场部门制定品牌传播策略
# 3. 高管层评估产品定位和定价策略

# 4. 如需补充调研，重新启动特定阶段：
hiclaw chat luxury-car-research/moderator
> 针对"后排老板座舒适度"进行补充辩论，组织三位富豪深度讨论
```

## 5. 预期效果

### 效率提升
- **时间**：从 3-6 个月缩短到 1-2 天（三阶段约 4-6 小时）
- **成本**：从 50-100 万元降低到 < 3 万元（主要是 API 调用费用）

### 质量提升
- **深度**：三阶段混合模式（独立陈述 + 交叉辩论 + 共识确认）> 传统焦点小组
- **广度**：覆盖美国/欧洲/中国三大市场的文化差异
- **真实性**：AI 角色扮演消除"社交期望偏差"，自由互动激发真实观点碰撞

### 决策支持
- **用户画像**：清晰的超高净值人群心理模型
- **需求矩阵**：量化的功能优先级排序
- **设计指南**：可直接落地的产品设计建议

## 7. 注意事项

### 技术风险
- **Token 消耗**：三阶段预计消耗 200K-400K tokens（比原 100 轮设计节省 50%），需提前充值
- **超时风险**：阶段 1 每个议题设置 10 分钟超时，阶段 2 每个议题 15 分钟超时
- **PDF 生成**：确保 Leader 环境安装了 pandoc 和中文字体

### 业务风险
- **AI 幻觉**：虽然角色扮演真实，但不能完全替代真实用户访谈
- **文化刻板印象**：需要人工审核，避免过度简化文化差异
- **决策依赖**：调研结果应作为参考，而非唯一决策依据

### 优化建议
- **动态议题**：根据阶段 1 的讨论热点，动态调整阶段 2 的辩论议题
- **人类介入**：在阶段 2 辩论时，产品总监可以加入提问
- **多轮迭代**：完成三阶段后，针对关键争议点再进行补充辩论

## 8. 总结

本案例展示了如何用 HiClaw 智能体团队替代传统市场调研流程，通过 **三阶段混合模式**（独立陈述 + 交叉辩论 + 共识确认）实现：

1. **高效**：1-2 天完成传统 3-6 个月的工作
2. **深度**：自由互动和辩论，远超传统焦点小组的机械问答
3. **灵活**：动态控制议题推进，而非机械计数轮次

**核心创新**：
- **Broadcast-Response**：阶段 1 并行收集观点，提升效率
- **Debate**：阶段 2 自由辩论，允许 Agent 之间直接 @mention 对话
- **Hierarchical**：阶段 3 层级确认，收敛最终共识

适用于所有需要"多角色深度讨论"的场景：
- 产品设计调研
- 品牌战略研讨
- 投资决策分析
- 危机公关推演
